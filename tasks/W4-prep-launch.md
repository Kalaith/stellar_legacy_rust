# W4 — Pre-launch PREP phase + explicit LAUNCH + provisioning

**Prerequisite: W7 complete and green.**

## Goal

Departure becomes a felt moment. In port the player **selects a charter**, provisions
the ship (food, spare parts, **fuel**), and commits with an explicit **[ LAUNCH ]**
button — replacing today's silent contract start. Under-provisioning has teeth: **no
food → starvation deaths** (exists), **no fuel → systems shut down and the ship may not
reach its destination** (new).

## Binding owner decisions

- Explicit prep → LAUNCH; accepting a charter must no longer silently start it.
- Provisioning: food/parts/fuel sized against the chosen voyage length. Food shortage
  kills; fuel shortage stalls travel and shuts systems down. Shortage-driven
  *opportunity events* (garden-world stop) are W6 content — W4 only builds the
  mechanics they need.
- Stay length is fixed by the charter (W2); prep offers no phase sliders.
- New game founder choice = the W7 faction picker (already built). Continuing voyages
  keep their people; Recruit verbs remain the way to add population/crew.
- All tunables in `assets/data/game_config.json`.

## Current state (verified facts)

- `UiAction::AcceptContract(id)` (`src/game/actions.rs:154`) immediately calls
  `contract::start_contract` — **this is the silent start being removed.**
- Port = `sim.contract.is_none()`; port-only gate pattern in `Game::purchase_component`.
- Fuel: `ShipState.fuel` is a 0–1 fraction; regenerated by engine loadout
  (`ship::apply_loadout_effects`, config `ship.fuel_regen_per_point`). Nothing consumes
  it during travel today.
- Food upkeep + starvation already exist in the year-boundary tick
  (`src/simulation/tick.rs`): shortage kills 2%/yr (medic-mitigated).
- Market Buy/Sell verbs exist for food/minerals/energy/influence
  (`src/simulation/market.rs`); spare parts restock via full repair.
- Capture harness scenes: `src/game/capture_scenes.rs::begin_capture_scene` maps scene
  names to composed states.
- Contract phases + `phase_index` exist (W2); factions + picker exist (W7).

## Changes

### 1. Charter selection without starting (`src/game/actions.rs`, `src/state/sim.rs`)

- `SimState` gains `pub selected_charter: Option<String>` (port-time only; cleared on
  launch).
- Rename `UiAction::AcceptContract` → `UiAction::SelectCharter(String)`: validates
  renown gate exactly as today, but only sets `selected_charter` and logs
  "Charter under consideration: {name}". Re-selecting swaps it.
- New `UiAction::Launch`:
  - requires port + `selected_charter` valid;
  - starts the contract (`contract::start_contract`), clears `selected_charter`,
    starts the mission timer (`mission_started`), logs
    "LAUNCH. {name} — {duration} years. May the line hold.";
  - **launch locks the loadout** — already true (port gates), no extra work beyond the
    gate staying `contract.is_none()`.
- The Contract screen becomes the **PREP** screen when a charter is selected: show the
  charter's phase plan (from W2 data), a provisioning readout (see step 3), and the
  [ LAUNCH ] button. Pure view + `UiAction`s, existing widget style.

### 2. Fuel as a consumable voyage store

Config additions (`GameConfig.ship` block or a new `provisioning` block — keep JSON
authoritative):

```json
"provisioning": {
  "fuel_burn_per_travel_month": 0.0035,
  "fuel_cost_credits_per_point": 30,
  "no_fuel_decay_multiplier": 2.0
}
```

- During **Travel** phase months (contract active, current phase kind Travel), burn
  `fuel_burn_per_travel_month` in the monthly loop. Operation months burn nothing.
- **Fuel at 0 during Travel:** the ship coasts — that month adds **no**
  `months_elapsed` progress toward the phase boundary (the voyage stretches; log once
  per year, not per month), and hull/life-support decay that year is multiplied by
  `no_fuel_decay_multiplier` ("systems shut down"). Population aging/economy continue
  — time still passes. This is the "may not reach the destination" failure: a dry tank
  strands the ship until fuel events/regen rescue it or the voyage dies.
  Implementation note: track stalled months so `target_duration_years` comparisons use
  contract months, not calendar months (W2 made the contract month-based; stalling
  simply doesn't advance `months_elapsed`).
- Port refuel verb: `UiAction::Refuel` — buys back to 1.0 at
  `fuel_cost_credits_per_point` × missing fraction × 100 (integer credits). Port-gated.
- Engine `fuel_regen` keeps working underway (that's the slow rescue path).

### 3. Provisioning readout (PREP screen, pure view)

For the selected charter, compute and display estimated needs vs. current stores:

- food: `population.count × food_per_person_per_year × duration_years` vs
  `resources.food` (plus a note that production offsets it — display the net yearly
  food balance from current production);
- spare parts: `parts_upkeep_per_year × duration_years` vs `ship.spare_parts`;
- fuel: `fuel_burn_per_travel_month × total_travel_months` vs `ship.fuel` (+regen note).

Color-code shortfalls with the existing warning styling. **No new purchase systems** —
provisioning uses the existing Market (food), repair restock (parts), and Refuel verbs.
The readout is what makes under-provisioning a visible choice.

### 4. Capture scene

Add a `"prep"` scene to `begin_capture_scene` (`src/game/capture_scenes.rs`): a port
sim with a charter selected and mixed provisioning levels, showing the PREP screen.
Keep existing scenes working.

### 5. Tests

- Selecting a charter does not start it; `Launch` does; `Launch` without a selection or
  underway is a no-op with a warning.
- Renown gating still enforced at selection time.
- Fuel burns only in Travel months; a zero-fuel Travel month adds no contract progress
  and applies the decay multiplier at the year boundary; refuel is port-only and charges
  correctly.
- A deliberately under-fueled autoplay run takes measurably longer (calendar months >
  contract months) — assert stalled months > 0 on a fixed seed.
- Autoplay harness updated: policy now selects charter + launches explicitly, refuels
  and provisions in port; W1/W2/W7 soak invariants stay green.

## Acceptance criteria

- All verification commands green.
- No code path starts a contract except `UiAction::Launch`.
- A fully-provisioned 340-yr autoplay run completes; a zero-fuel-at-launch run visibly
  stalls (and may die) — both deterministic per seed.
- `.\scripts\capture_ui.ps1 -Scenes menu,prep,gameplay` produces a PREP screenshot
  (run it if the script exists in this project; otherwise
  `..\macroquad-toolkit\scripts\capture_ui.ps1 -Scenes prep` from the project dir).

## Ground rules

1. Data-driven: every new number in `game_config.json`.
2. 800-line hard limit; the PREP screen goes in its own `src/ui/prep.rs`
   (`pub mod prep;` in `ui.rs`) rather than growing an existing panel.
3. UI is a pure view; PREP emits `SelectCharter`/`Launch`/`Refuel` only.
4. Determinism: randomness only through `sim.rng`.
5. Old saves abandoned; no migration shims.
6. Delete unused code outright (the old `AcceptContract` naming and any dead paths).
